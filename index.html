Django wsgi源码学习

Posted on 2018-04-23 | In Django

Django 源码分析 1.11

在Web应用启动后，django.setup()加载settings,配置logging,加载app！
之后会生成一个 WSGIHandler 实例,也就是WSGI application接口实现的可调用对象
在实例化WSGIHandler的__init__方法中,完成加载中间件
setting中的WSGI_APPLICATION = ‘project.wsgi.application’ 与此配置相关
之后每次由WSGI server转发来的请求都会使用这个实例进行响应(当然也可能同时存在多个application实例:与WSGI服务器的配置有关)

对于每次请求,WSGI server调用WSGIHandler的__call__方法

  • 1.生成WSGIRequest实例,发送request_started信号
  • 2.进入中间件的链式调用,loops through the request methods, running them in order
  • 3.resolves the requested URL 处理url
  • 4.loops through each of the view processing methods
  • 5.calls the view function (usually rendering a template),也就是url模块中的as_view
  • 6.processes any exception methods
  • 7.loops through each of the response methods, (from the inside out, reverse order from request middlewares) 注意：这里的顺序是逆序
  • 8.calls the callback function to the web server

python的相对引用

Posted on 2017-12-29 | In python

python -m 参数

python -m参数 有错误

  • python dir/file.py 直接运行（将file.py的__name__属性设置’__main__‘)
  • python dir.file 以模块的形式启动（这里的dir目录下必须包含__init__.py)

会影响python进程的sys.path属性

  • 直接启动是把test.py文件所在的目录放到了sys.path属性中
  • 而模块启动是把你输入命令的目录（也就是当前路径），放到了sys.path属性中。

两种启动方式中 __name__值的不同

  • 直接启动, __name__ == __main__
  • 模块启动, __name__ == file(文件名,和包的继承关系有关,module.module3.filename)

注意：不管各种启动方式,作为python解释的第一个文件,其__name__属性必然是__main__,只有作为模块被其他py文件引用时的该属性才会体现包继承关系
