NAME
    time - This module provides various functions to manipulate time values.

DESCRIPTION
    There are two standard representations of time.  One is the number
    of seconds since the Epoch, in UTC (a.k.a. GMT).  It may be an integer
    or a floating point number (to represent fractions of seconds).
    The Epoch is system-defined; on Unix, it is generally January 1st, 1970.
    The actual value can be retrieved by calling gmtime(0).

    The other representation is a tuple of 9 integers giving local time.
    The tuple items are:
      year (including century, e.g. 1998)
      month (1-12)
      day (1-31)
      hours (0-23)
      minutes (0-59)
      seconds (0-59)
      weekday (0-6, Monday is 0)
      Julian day (day in the year, 1-366)
      DST (Daylight Savings Time) flag (-1, 0 or 1)
    If the DST flag is 0, the time is given in the regular time zone;
    if it is 1, the time is given in the DST time zone;
    if it is -1, mktime() should guess based on the date and time.

继承Thread类，override父类的__init__和run方法

The Thread class represents an activity that is run in a separate thread of control. There are two ways to specify the activity: by passing a callable object to the constructor, or by overriding the run() method in a subclass. No other methods (except for the constructor) should be overridden in a subclass. In other words, only override the __init__() and run() methods of this class.

join和setDaemon

主线程结束后，子线程还在运行，join方法使得主线程等到子线程结束时才退出。
ps：主线程结束，进程未结束，join方法定义了线程结束的顺序，主线程必须在定义了join方法的子线程结束之后再结束

设置守护线程threading.Thread().setDaemon(bool值)
默认位False，所有线程结束之后进程退出。对于setDaemon（True)的线程，进程不再等待该线程正常结束，其他非守护线程结束之后不关心守护线程是否执行完毕，直接退出进程，结束程序。
必须在线程start之前设置

线程间数据共享

资源锁 Lock and RLock

在线程间共享多个资源的时候，如果两个线程分别占有一部分资源并且同时等待对方的资源，就会造成死锁
避免死锁，使用递归锁（可重用锁）

threading.RLock() 类内部有一个计数器，acquire和release必须成对出现，即调用了n次acquire，必须调用n次的release才能真正释放所占用的琐。

多个线程调用main()方法，而main方法在内部调用的函数也使用了锁，有多层锁，只有一个线程中的所有锁都释放，共享数据才能被其他线程使用。

信号量（Semaphore）

并行锁（区别于RLock的递归）,
threading.BoundedSemaphore(value=1) 提供类似线程池的并行锁

条件变量同步（Condition）

有一类线程需要满足条件之后才能够继续执行，Python提供了threading.Condition 对象用于条件变量线程的支持，它除了能提供RLock()或Lock()的方法外，还提供了 wait()、notify()、notifyAll()方法。

  • wait()：条件不满足时调用，线程会释放锁并进入等待阻塞；
  • notify()：条件创造后调用，通知等待池激活一个线程；（当有多个线程wait时，激活线程随机，–待验证）
  • notifyAll()：条件创造后调用，通知等待池激活所有线程。

同步条件（Event）

条件同步和条件变量同步差不多意思，只是少了锁功能，因为条件同步设计于不访问共享资源的条件环境。初始设计值为False

  • event.isSet()：返回event的状态值；
  • event.wait()：如果 event.isSet()==False将阻塞线程；
  • event.set()： 设置event的状态值为True，所有阻塞池的线程激活进入就绪状态， 等待操作系统调度；
  • event.clear()：恢复event的状态值为False。

python的相对引用

python -m 参数

python -m参数 有错误

  • python dir/file.py 直接运行（将file.py的__name__属性设置’__main__‘)
  • python dir.file 以模块的形式启动（这里的dir目录下必须包含__init__.py)

会影响python进程的sys.path属性

  • 直接启动是把test.py文件所在的目录放到了sys.path属性中
  • 而模块启动是把你输入命令的目录（也就是当前路径），放到了sys.path属性中。

两种启动方式中 __name__值的不同

  • 直接启动, __name__ == __main__
  • 模块启动, __name__ == file(文件名,和包的继承关系有关,module.module3.filename)

注意：不管各种启动方式,作为python解释的第一个文件,其__name__属性必然是__main__,只有作为模块被其他py文件引用时的该属性才会体现包继承关系

join 和 task_done

ps：task_done中的任务是指get任务，put方法不需要调用
